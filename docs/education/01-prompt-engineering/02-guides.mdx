# Prompt Elements

<table>
  <thead>
    <tr>
      <th>Element</th> <th>Description</th> <th>Impact</th>
      <th>General Examples</th> <th>Tech/IT Examples</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <b>Clear &amp; Specific Task/Instruction</b>
      </td>
      <td>
        The core action you want the AI to perform. This should be an
        unambiguous and direct command or question. Use action verbs and avoid
        vagueness
      </td>
      <td>
        This is the fundamental directive for the AI. Without a clear task, the
        model has no direction and is likely to produce irrelevant or generic
        output
      </td>
      <td>
        <ul>
          <li>Summarize this article</li>
          <li>
            Generate five headlines for a blog post about sustainable living
          </li>
          <li>Translate the following paragraph into French</li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            Write a Python function to sort a list of dictionaries by a specific
            key
          </li>
          <li>
            Explain how blockchain technology works to a non-technical audience
          </li>
          <li>Debug the following Python code snippet</li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Context</b>
      </td>
      <td>
        The background information, setting, or circumstances relevant to the
        task. This helps the AI understand the "why" behind your request
      </td>
      <td>
        Context narrows the AI's focus, allowing it to generate a more relevant
        and tailored response that aligns with your specific situation or need
      </td>
      <td>
        <ul>
          <li>
            I am writing a follow-up email to a client I met at a conference.
            They expressed interest in our new software
          </li>
          <li>
            I need help planning a road trip through national parks in the
            Southwest United States for next summer
          </li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            I'm developing a new microservice using FastAPI and I need to
            implement secure API authentication
          </li>
          <li>
            We are experiencing slow database query performance in our
            PostgreSQL cluster during peak hours
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Expectations</b>
      </td>
      <td>
        Clearly defining what a "good" response looks like including
        completeness, level of detail, or aspects to emphasize or avoid
      </td>
      <td>
        Helps the AI align more closely with success criteria, reducing
        trial-and-error by anticipating how the output will be evaluated or used
      </td>
      <td>
        <ul>
          <li>
            Make sure the summary covers the main arguments and key takeaways
          </li>
          <li>Provide at least three different approaches</li>
          <li>Do not include any personal opinions</li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            Ensure the code is well-commented and follows best practices for
            readability and maintainability
          </li>
          <li>
            Explain the advantages and disadvantages of each encryption method
          </li>
          <li>
            The solution should not introduce any new security vulnerabilities
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Role/Persona</b>
      </td>
      <td>
        Assigning a specific role or persona for the AI to adopt (e.g., "Act as
        a seasoned financial analyst," "You are a creative copywriter")
      </td>
      <td>
        This significantly influences the tone, style, vocabulary, and even the
        reasoning process of the AI, leading to more authentic and useful
        outputs
      </td>
      <td>
        <ul>
          <li>
            Act as a seasoned financial analyst and evaluate the potential ROI
            of investing in renewable energy stocks
          </li>
          <li>
            You are a motivational coach; write a short pep talk for someone
            starting a new fitness journey
          </li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            Act as a senior DevOps engineer advising on cloud infrastructure
            migration strategies
          </li>
          <li>
            You are a cybersecurity expert; outline the potential attack vectors
            for a new mobile application
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Audience</b>
      </td>
      <td>
        Specifying the intended audience for the AI's response (e.g., "Explain
        this to a 5-year-old," "Write for a technical audience")
      </td>
      <td>
        This dictates the level of complexity, language, and detail in the
        response, ensuring it is understandable and impactful for the target
        reader
      </td>
      <td>
        <ul>
          <li>Explain quantum physics to a 5-year-old</li>
          <li>
            Write a technical report on the latest advancements in AI for an
            audience of machine learning engineers
          </li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            Explain the concept of containerization to a project manager with no
            technical background
          </li>
          <li>
            Write documentation for API endpoints for an audience of third-party
            developers
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Format</b>
      </td>
      <td>
        Defining the desired structure of the output (e.g., "in a bulleted
        list," "as a JSON object," "in a table with columns for...")
      </td>
      <td>
        A specified format ensures the information is organized in a usable and
        easily digestible manner, saving time on post-processing
      </td>
      <td>
        <ul>
          <li>Provide the information in a bulleted list</li>
          <li>
            Output the data as a JSON object with keys for 'name' and 'age'
          </li>
          <li>
            Create a table with columns for 'Product Name', 'Price', and
            'Availability'
          </li>
        </ul>
      </td>
      <td>
        <ul>
          <li>Generate the YAML configuration for a Kubernetes deployment</li>
          <li>Provide the error logs in a structured JSON format</li>
          <li>
            Present a comparison of SQL and NoSQL databases in a table with
            columns for 'Feature', 'SQL', and 'NoSQL'
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Source</b>
      </td>
      <td>
        Providing authoritative references or datasets the AI should base its
        response on, or citing where information should come from.
      </td>
      <td>
        Anchors the AI's response in credibility or required material,
        especially critical for factual accuracy, traceability, or domain
        compliance
      </td>
      <td>
        <ul>
          <li>
            Using the provided research paper, summarize the methodology section
          </li>
          <li>
            Referencing the attached sales data, identify the top 3 performing
            regions
          </li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            Based on the OpenAPI specification provided, generate client-side
            code in Java
          </li>
          <li>
            Analyze the attached network traffic logs and identify any anomalous
            activity
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Examples (Few-Shot Prompting)</b>
      </td>
      <td>
        Providing one or more examples of the desired input-output pattern
      </td>
      <td>
        This is a powerful way to guide the AI's response, especially for
        complex or novel tasks, by demonstrating the expected style and
        structure
      </td>
      <td>
        <ul>
          <li>Input: 'Happy' -> Output: 'Joyful, Elated'</li>
          <li>Input: 'Sad' -> Output: 'Mournful, Downcast'</li>
          <li>Now, Input: 'Angry' -> Output:...</li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            Example Input: `def add(a, b): return a + b` -> Example Output:
            `Function: add, Parameters: a, b, Description: Adds two numbers.`
          </li>
          <li>
            Example Input: `class User:` -> Example Output: `Class: User,
            Description: Represents a user.`
          </li>
          <li>Now, Input: `import os` -> Output:...</li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Constraints &amp; Negative Constraints</b>
      </td>
      <td>
        Setting boundaries or limitations on the response (e.g., "in under 200
        words," "do not include technical jargon," "avoid using passive voice")
      </td>
      <td>
        Constraints help to refine the output, making it more focused and
        aligned with specific requirements. Negative constraints are
        particularly useful for avoiding undesirable content
      </td>
      <td>
        <ul>
          <li>Summarize the article in under 200 words</li>
          <li>Do not include any technical jargon in your explanation</li>
          <li>Ensure the response avoids using passive voice</li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            Generate a SQL query that runs in under 100ms on a database with 1
            million records
          </li>
          <li>Do not use any deprecated libraries in the code example</li>
          <li>
            Avoid suggesting solutions that require significant infrastructure
            changes
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Tone &amp; Style</b>
      </td>
      <td>
        Describing the desired emotional or stylistic quality of the response
        (e.g., "formal," "humorous," "empathetic," "optimistic")
      </td>
      <td>
        Tone and style add a layer of personality and nuance to the output,
        making it more engaging and appropriate for the context
      </td>
      <td>
        <ul>
          <li>Write a formal business email</li>
          <li>Craft a humorous short story about a talking cat</li>
          <li>Provide an empathetic response to a customer complaint</li>
        </ul>
      </td>
      <td>
        <ul>
          <li>Write a concise and professional bug report</li>
          <li>Craft an engaging blog post about the future of AI</li>
          <li>
            Explain a complex technical concept in a casual and approachable
            manner
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Chain of Thought/Step-by-Step Reasoning</b>
      </td>
      <td>
        Instructing the AI to "think step-by-step" or outline its reasoning
        process before providing the final answer
      </td>
      <td>
        This technique can improve the accuracy of the final output, especially
        for complex reasoning tasks, and provides transparency into the AI's
        process
      </td>
      <td>
        <ul>
          <li>
            Think step-by-step to solve this math problem and show your work
          </li>
          <li>
            Before providing the final answer, explain your reasoning process
            for choosing the best solution
          </li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            Walk me through the steps of deploying a Docker container to AWS ECS
          </li>
          <li>
            Before suggesting a solution, analyze the provided code for
            potential performance bottlenecks and explain your findings
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Questioning &amp; Iteration</b>
      </td>
      <td>
        Asking clarifying questions within the prompt or planning for iterative
        refinement of the AI's responses
      </td>
      <td>
        This fosters a more collaborative interaction, allowing you to guide the
        AI towards a better answer through a series of prompts
      </td>
      <td>
        <ul>
          <li>
            If you need more information about the target audience, what
            specific details would be most helpful?
          </li>
          <li>
            Based on this initial draft, what areas do you think need further
            refinement?
          </li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            What specific operating system or cloud provider are you targeting
            for this deployment?
          </li>
          <li>
            Could you provide the exact error message you're receiving, or any
            relevant stack traces?
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Request for Multiple Versions</b>
      </td>
      <td>
        Asking for several different versions or perspectives on the same topic
      </td>
      <td>
        This can be useful for brainstorming, exploring different creative
        angles, or A/B testing different approaches
      </td>
      <td>
        <ul>
          <li>
            Provide three different taglines for a new coffee shop, each with a
            different tone
          </li>
          <li>
            Generate two distinct summaries of the article: one concise and one
            detailed
          </li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            Generate three different architectural diagrams for a scalable web
            application, highlighting different technology stacks
          </li>
          <li>
            Provide two alternative approaches to implementing user
            authentication, one using OAuth and another using JWT
          </li>
        </ul>
      </td>
    </tr>
    <tr>
      <td>
        <b>Inclusion of Keywords</b>
      </td>
      <td>
        Strategically placing important keywords within the prompt that you want
        to see in the output
      </td>
      <td>
        This can help to ensure that the AI's response focuses on the most
        critical aspects of your request
      </td>
      <td>
        <ul>
          <li>
            Write a press release about our new product launch, ensuring to
            include
          </li>
          <li>
            'innovation', 'sustainability', and 'customer-centric' in the text
          </li>
        </ul>
      </td>
      <td>
        <ul>
          <li>
            When describing the microservices architecture, ensure to use terms
          </li>
          <li>
            like 'loose coupling', 'API gateway', and 'container orchestration'
          </li>
          <li>
            The security audit report should emphasize 'vulnerability
            assessment', 'penetration testing', and 'compliance standards'
          </li>
        </ul>
      </td>
    </tr>
  </tbody>
</table>
